# Presentation assets — OG-Core × CLEWS integration framework

A focused set of assets for presenting the OG-Core ⇄ CLEWS/OSeMOSYS soft-link. The one
live deliverable is the **mechanisms deck** (`mechanisms.tex` → `mechanisms.pdf`): how the
two models connect — the economic channels and how each is represented in the model.
Everything shares one editorial, colorblind-safe palette
([`theme/ogclews-colors.tex`](theme/ogclews-colors.tex), mirrored from `ogclews_link/style.py`).

Numbers in the deck are the Philippine reference run (`results/golden.json` + the solved
steady states), verified against the code. The plain-language / audience-facing voice
deliberately avoids code identifiers and terms of art (no "dual" — it says "the system's
own marginal cost"; equations are fine, but every symbol is paired with plain words).

## Build

```bash
bash build.sh        # diagrams -> PDFs, then compile mechanisms.pdf
```

Requires TeX Live (`pdflatex` + `latexmk`). The deck embeds three committed golden figures
from `../results/figures/*.png` (regenerated by the model, not here). Build outputs (`*.pdf`)
are git-ignored and regenerated by `build.sh` — except `mechanisms.pdf`, which is committed
so the deck is viewable in-repo without a local TeX build.

## Reusing the assets

- **Drop a diagram into your own slides / paper:** `\includegraphics{diagrams/architecture.pdf}`.
  Each diagram is a `standalone` document — edit the `.tex` and recompile just that file
  (`cd diagrams && latexmk -pdf architecture.tex`).
- **Match the colours** in new material by `\input{theme/ogclews-colors}` and using
  `ogc` (OG-Core blue), `clews` (teal), `policy` (red), `claret`, `ink`/`sub`/`mute`.

The diagrams are content-faithful to the code (`ogclews_link/channels.py`). If the channels
change, update the diagrams to match — see [`../paper`](../paper) for the formal write-up.
